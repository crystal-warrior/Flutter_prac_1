## 📖 О проекте

**GardenHelper** — это полнофункциональное Flutter-приложение для садоводов и дачников, которое помогает планировать посадки, следить за погодой, получать рекомендации по уходу за растениями и управлять своим садовым календарем.

Ты не просто садовод — ты профессионал с умным помощником.

⚡ Приложение разработано на **Flutter** с использованием **Clean Architecture** и современных практик разработки.

---

## 🚀 Возможности

### 🌤️ Погода и климат
- 📍 **Точные прогнозы погоды** от Yandex.Weather API
- 🌍 **Сравнение погоды в разных регионах** для планирования посадок
- 📊 **Детальная информация**: температура, влажность, давление, скорость ветра
- 🗓️ **Прогноз на несколько дней** с почасовыми данными

### 📅 Календарь посадок
- 📝 **Персональный календарь** с заметками о посадках
- 💾 **Локальное хранение** в SQLite с синхронизацией
- 📱 **Красивый интерфейс** с календарным видом

### 🌙 Лунный календарь
- 🌑 **Фазы луны** для оптимального времени посадок
- 📆 **Рекомендации по дням** для разных культур
- 🔮 **Точные расчеты** лунных фаз

### 🌍 Геолокация и карты
- 🗺️ **Геокодирование адресов** через Yandex.Maps API
- 📍 **Определение местоположения** по IP-адресу
- 🏡 **Сохранение адресов** садовых участков
- 🔍 **Поиск по адресу** с автодополнением

### 💧 Уход за растениями
- 💧 **Регуляция полива** с персональными рекомендациями
- 🌿 **Советы по уходу** за различными культурами
- 🌱 **Мои растения** — каталог ваших культур
- 🌾 **Рекомендуемые растения** для вашего региона

### 👤 Профиль и настройки
- 🔐 **Безопасная аутентификация** с токенами
- 🎨 **Темная/светлая тема** оформления
- 📝 **Управление профилем** с данными о местоположении
- 🔒 **Безопасное хранение** токенов в Secure Storage

### 🌅 Дополнительно
- ☀️ **Восход и закат** солнца для вашего региона
- 📊 **Статистика** по посадкам и уходу

## 🛠️ Технологии

### Клиент
- **Flutter 3.9.2+** — кроссплатформенная разработка
- **Dart 3.9.2+** — современный язык программирования
- **BLoC/Cubit** — управление состоянием
- **GoRouter** — навигация и маршрутизация
- **GetIt** — dependency injection
- **Retrofit + Dio** — работа с REST API
- **JSON Serialization** — автоматическая сериализация

### Локальное хранение
- **SQLite (sqflite)** — база данных для календаря посадок
- **SharedPreferences** — настройки и пользовательские данные
- **Flutter Secure Storage** — безопасное хранение токенов

### Геолокация
- **Geolocator** — определение местоположения
- **Yandex.Maps Geocoding API** — геокодирование адресов
- **IP Geolocation API** — определение региона по IP

### Архитектура
- **Clean Architecture** — разделение на слои (UI, Domain, Data)
- **Repository Pattern** — абстракция источников данных
- **Use Case Pattern** — бизнес-логика
- **DTO Pattern** — передача данных между слоями

## 🔌 API Интеграции

### Yandex.Weather API
- 🌡️ **Текущая погода** по координатам
- 📍 **Погода по городу** с геокодированием
- 🔄 **Сравнение погоды** в нескольких регионах
- 📊 **Прогноз на несколько дней** с детальной информацией

### Yandex.Maps Geocoding API
- 🔍 **Поиск по адресу** → координаты
- 📍 **Обратное геокодирование** (координаты → адрес)
- 🏙️ **Поиск по городу** для получения координат
- 📋 **Массовое геокодирование** нескольких адресов
- 🗺️ **Определение адресов** сохраненных мест

### IP Geolocation API
- 🌍 **Определение региона** по IP-адресу
- 📍 **Автоматическое определение** местоположения при запуске

### OpenWeatherMap API
- 🔄 **Резервный источник** данных о погоде
- 🌐 **Международная поддержка** регионов

## 🏗️ Архитектура

Приложение построено на принципах **Clean Architecture** с четким разделением ответственности:

```
┌─────────────────────────────────────┐
│         UI Layer (Features)         │
│  ┌─────────┐  ┌─────────┐           │
│  │ Screens │  │  Cubit  │           │
│  └─────────┘  └─────────┘           │
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│      Domain Layer (Business)        │
│  ┌──────────┐  ┌──────────────┐     │
│  │ UseCases │  │ Repositories  │    │
│  └──────────┘  └──────────────┘     │
└───────────────┬─────────────────────┘
                │
┌───────────────▼─────────────────────┐
│        Data Layer (Storage)         │
│  ┌──────────┐  ┌──────────────┐     │
│  │DataSources│ │ Repositories │     │
│  └──────────┘  └──────────────┘     │
└─────────────────────────────────────┘
```

### Принципы
- ✅ **Разделение ответственности** — каждый слой решает свои задачи
- ✅ **Инверсия зависимостей** — зависимости направлены внутрь
- ✅ **Тестируемость** — легко подменять реализации
- ✅ **Масштабируемость** — простое добавление новых фич

## 🔐 Безопасность

- 🔒 **Токены авторизации** хранятся в Flutter Secure Storage
- 🚫 **Пароли не сохраняются** — используются только при аутентификации
- 🔑 **Автоматическое обновление токенов** через refresh token
- ⏰ **Валидация токенов** с проверкой срока действия (7 дней)
- 🛡️ **Шифрование данных** на уровне ОС (Android/iOS)
